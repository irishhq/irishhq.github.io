vue 移动端真机调试

  1. 前提：程序运行的本机和手机在同一网络环境下
  2. 查询vue移动端项目和后台接口项目运行所在本地ip地址，例如本机ipv4地址为：http://192.168.1.9
  3. 修改后台接口程序的监听地址为查询所得ipv4的地址
    vue中配置文件位置：src->common->js->config.js
    export const HOST = 'http://192.168.1.9:3000'

  4. 配置vue中config->index.js(项目主要配置文件，包括监听端口，打包路径等)
    ->修改开发环境配置

  5. 手机浏览器打开上述的ipv4网址，即可看到页面。

cordova打包vue项目为Android应用（apk文件）

前提：一个用vue-cli搭建的一个vue移动端项目

  1. 所需的环境(验证方式不唯一)
    • Java环境
      • 验证环境是否配置好：java和javac
      • 配置：
        1. 下载JDK: 最好就默认安装，安装后目录类似于:C:\Program Files\Java下包含两个文件夹jdk1.8.0_201`jre1.8.0_201`
        2. 环境变量配置
          系统变量->JAVA_HOME->C:\Program Files\Java\jdk1.8.0_201(jdk安装目录)
          系统变量->CLASSPATH->.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;
          用户变量->Path新增->%JAVA_HOME%\bin
          用户变量->Path新增->%JAVA_HOME%\jre\bin
    • Android环境
      • 可以直接下载androidSDK
        也可以用Android Studio的SDK Manager下载安装SDK Platforms和SDK Tools,这种方式比较省事
      • 配置环境：
        系统变量->ANDROID_HOME->C:\android-sdk-windows(安卓SDK下载的位置)
        环境变量->Path新增->%ANDROID_HOME%\platform-tools
        环境变量->Path新增->%ANDROID_HOME%\tools
        环境变量->Path新增->%ANDROID_HOME%\build-tools\28.0.0
        ANDROID_HOME和JAVA_HOME也可以不配置，直接用完整的地址代替
    • cordova
      • 验证cordova -v
      • 安装npm install -g cordova --registry=https://registry.npm.taobao.org
      • 验证AndroidSDK和Android target 是否正确安装cordova requirements
  2. 创建cordova项目
    cordova create cordovaAppDirName packName projectName
  3. 添加安卓平台
    进入cordovaAppDirName,
    cordova platform add android
  4. 将vue项目置于cordovaAppDir根目录下

  5. 修改Vue项目中config->index.js

  6. 编译vue项目
    在vue项目根目录下npm run build
    此时会在cordovaAppDir下的www文件夹中生成项目

  7. 打包apk:
    cordovaAppDir根目录下运行cordova build android生成apk文件
